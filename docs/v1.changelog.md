### [1.016.000] - 2026-02-21  
## Cryptographic Trust Layer, Multi-Key Rotation, Audit Chain, CI Enforcement

This release establishes a cryptographically verifiable, tamper-evident payroll integrity architecture.

### Ed25519 Detached Signatures
- Canonical payroll verification payload signed server-side using Ed25519 (libsodium).
- Client-side verification implemented using tweetnacl/libsodium.js.
- Private signing keys stored securely on server file system (not in `.env`).
- No shared-secret (HMAC) logic remains in system.

### Canonical Scoped Verification Payload
- Deterministic, ordered JSON payload including:
  - `scope = pay_period`
  - `period.start`, `period.end`, `frequency`
  - `engineVersion`
  - `bracketVersion`
  - `signingKeyVersion`
  - `grossCents`, `taxCents`, `netCents`
- Contract frozen via snapshot test.
- Any structural or serialization drift fails CI.

### Append-Only Audit Chain
- Chain continuity defined as: `chainHash_n = SHA256(previousChainHash + currentPayloadHash)`
- Tampering with historical payloads invalidates downstream chain.
- Removal or mutation of any period breaks verification.
- Standalone forensic chain validator implemented (CLI).

### Multi-Key Rotation & Revocation
- Versioned signing keys (`SIGNING_KEY_VERSION`).
- Mixed-version historical verification supported.
- Revoked keys blocked from signing but retained for historical verification.
- Rotation simulation harness validates lifecycle transitions.

### CLI Cryptographic Lifecycle Tooling
- `generate_keys.php` – versioned Ed25519 keypair generation.
- `simulate_rotation.php` – rotation, revocation, tamper simulation.
- `verify_chain.php` – standalone forensic audit-chain validator.
- All scripts exit non-zero on integrity failure.

### CI Enforcement
- Public-key pinning test (backend vs frontend key equality).
- Cryptographic lifecycle regression tests integrated into PHPUnit.
- Optional stress mode (`CRYPTO_STRESS=1`) for large-period simulations.

### Telemetry Enforcement
- Signature or chain verification failures POST structured telemetry.
- Backend increments counters and logs structured event context.
- Invariant: no silent integrity failure paths.

This release transitions PayCal from application-level trust to cryptographically verifiable payroll integrity with operational rehearsal and CI enforcement.
